# 超长篇故事级记忆系统

> 解决 v2.0 无法追踪跨越数百章的故事线、情绪弧、和并行叙事的问题

---

## 问题根源

当前系统用「卷」作为中层规划单位，但卷是**创作调度单位**（50章），不是**故事逻辑单位**。一条故事线可能跨越 3 个卷，但在当前系统里它没有自己的「文件」——它的状态散落在若干卷总结中，没有连续追踪。

```
当前层级:
  弧 (300章)  →  卷 (50章)  →  章
  ↑故事逻辑    ↑调度逻辑     ↑创作单位

缺失层级:
  弧 (300章)  →  篇章 (100-150章)  →  卷 (50章)  →  章
  ↑故事逻辑    ↑故事逻辑分组       ↑调度逻辑     ↑创作单位
               ↑新增此层！
```

**篇章（Part）** 是3-6个卷组成的、有独立主题的故事段落。例如：
- 篇章1「初入江湖」（卷1-4，第1-200章）
- 篇章2「宗门风云」（卷5-8，第201-400章）
- 篇章3「天下争霸」（卷9-14，第401-700章）

每个篇章有自己完整的：开头→发展→转折→高潮→结尾——形成一条完整的**微型故事弧**。

## 新增文件系统

在 v2.0 的 15 个文件基础上，新增以下 6 个文件：

### 1. 篇章规划 `part-plan.md`

每个弧拆为 3-5 个篇章，篇章是一段完整故事逻辑的容器。

```markdown
# 篇章规划

## 篇章1「初入江湖」（第1-200章，卷1-4）
- 主题：从平凡人到江湖人的转变
- 起点状态：平凡少年，不知江湖凶险
- 终点状态：懂得江湖规则，但还保持善良
- 核心冲突：理想vs现实（对江湖的美好想象被打破）
- 关键节点：
  | 事件 | 章节 | 角色情绪基准 |
  |------|:---:|------------|
  | 初遇江湖 | C0020 | 好奇+兴奋 |
  | 第一次背叛 | C0060 | 震惊+愤怒 |
  | 江湖规则课 | C0120 | 困惑→理解 |
  | 第一次杀人 | C0180 | 恐惧+成长 |
  | 篇章高潮：为友复仇 | C0200 | 决绝+空虚 |

## 篇章2「宗门风云」（第201-450章，卷5-9）
- 主题：权力的腐蚀
  ...

## 篇章3「天下争霸」（第451-700章，卷10-14）
  ...
```

### 2. 篇章总结 `part-summaries/part-01-summary.md`

篇章完成后生成完整叙事总结——不是调度层面的卷总结，而是**故事层面的叙述总结**。

```markdown
# 篇章1「初入江湖」故事总结
第1-200章，卷1-4，约 800,000 字

## 故事梗概
主角张三从青州小城走出，带着对江湖的美好幻想。在第一次遭遇
盗匪时，他天真的出手相助反而害死了一个无辜路人。这是他的第一课：
江湖不是侠客话本。随后的旅途中，他遇到了亦师亦友的前辈王老剑客，
王老教他武功和江湖规矩。但他在第120章发现，王老曾经也是他心目中
的那类「恶人」。这种幻灭比第一次背叛还痛。

第150章，他被迫第一次杀人——杀的是一个和他同龄的少年盗匪。
这件事在他心里留下了裂缝。但更大的冲击在篇章结尾：第180-200章，
他发现害死自己父母的凶手，竟然是王老曾经的弟子。他杀入仇人家中，
完成复仇。但复仇的「爽」只有一瞬——之后是巨大的空虚。

## 叙事线追踪

### 主线A：主角的江湖之旅
第1-200章 | 状态：已完成
情绪弧：天真好奇→幻灭→接受→愤怒→复仇→空虚
关键转折：C0020(第一个教训), C0060(背叛), C0120(王老真相), C0180(第一次杀人), C0200(复仇)

### 支线B：王老剑客的过去
第50-120章 | 状态：本篇章已完成揭示
情绪弧（读者视角）：崇拜→怀疑→理解→复杂
关键转折：C0050(王老出场), C0080(第一次怀疑), C0120(真相揭示)

### 支线C：父母之死的线索
第10-200章 | 状态：阶段性完成（凶手已死）
情绪弧：悬念→线索碎片→逼近真相→复仇→空虚
关键转折：C0010(遗物暗示), C0080(第一条线索), C0150(真凶浮现), C0200(复仇完成)
遗留：遗物的第三层含义尚未揭示（将在篇章3展开）

## 遗留问题（交给后续篇章）
1. 《天剑九式》剑谱的最后一页——王老临终前说「不要练最后一式」
2. 遗物玉佩在第200章复仇时碎了一半——另一半在哪？
3. 李四在第180章神秘消失后再未出现

## 读者体验评估
整体节奏：⭐⭐⭐⭐ 紧凑（每个5章段有明确的目的）
情绪丰富度：⭐⭐⭐⭐ 从兴奋到空虚，情绪区间完整
角色弧完成度：⭐⭐⭐⭐ 主角弧完成，王老弧完成，李四弧留到下一篇章
```

### 3. 故事线追踪文件 `storylines/`

每条叙事线一个独立文件，追踪其从开始到结束的完整路径。

```
storylines/
├── mainline-A.md          # 主线A：主角成长/主要冲突
├── subline-B-王老.md      # 支线B：王老剑客的过去
├── subline-C-父母之死.md  # 支线C：寻找父母之死的真相
├── subline-D-魔教.md      # 支线D：魔教的崛起
└── subline-E-散修联盟.md  # 支线E：散修联盟的兴衰
```

每条线的文件格式：

```markdown
# 支线C：寻找父母之死的真相
创建于：第10章（首次暗示）
状态：◉ 进行中（第2457章）
预期结束：第2700章（篇章5）
优先级：主线驱动（直接影响主角弧光）
叙事类型：悬念/解密

## 情绪弧（故事级别的读者情绪设计）
```
章节    C0010  C0080  C0150  C0200  ...  C2280  C2350  C2400  C2457  C2600  C2700
情绪    疑惑  期待  逼近  复仇后空虚  重新  希望  挫败  坚持   揭秘   释然
                                       追踪
强度    ██░░  ███░  ████  ███░  ██░░  ███░  ███░  ██░░  ███░   ████  ██░░
```

## 叙事路径
每一段都有：章范围 + 发生了什么 + 角色此时的状态

### 第一段：暗示与迷惘（C0010-C0079）
- 触发：第10章，主角在母亲遗物中找到半块玉佩
- 发展：第40章，一个老商贩看到玉佩后脸色大变，拒绝交谈
- 读者此时的状态：轻微悬念，不明所以
- 角色此时的状态：刚丧母，情感还在处理丧母之痛，玉佩只是不经意提到

### 第二段：第一条线索（C0080-C0149）
- 触发：第80章，王老无意间提到「这块玉佩的纹路...我见过」
- 发展：第100章，王老带主角去一个废弃的庙，找到了玉佩的另一半
- 发展：第120章，发现父母曾在这里被追杀
- 发展：第140章，真凶身份浮现——王老的前弟子
- 读者此时的状态：悬念升级，开始主动猜测
- 角色此时的状态：愤怒，但被王老按住

### 第三段：复仇（C0150-C0200）
- 第150章：追杀真凶
- 第180章：在仇人家中发现遗书的母本——父母死前写了什么？
- 第200章：复仇完成，但发现遗书还缺一页
- 读者此时的状态：满足（复仇完成）但留有好奇（缺的那页）
- 角色此时的状态：空虚，困惑（为什么复仇没有想象中满足）

### 第四段：第二层秘密（C2200-C2349）← 长篇中断后的重启
- ⚠️ 注意：这条线从第200章暂停，到第2200章才恢复！
  中间隔了2000章——agent在恢复这条线时需要完整读取此前所有段。
- 触发：第2200章，主角在魔教遗址发现了缺的那页遗书
- 发展：第2280章，遗书指向一个上古遗迹——父母不是意外死亡
- 读者此时的状态：2000章后猛然想起这条线，记忆需要唤醒
- 角色此时的状态：已从第200章的空虚中走出，现在被新的信息点燃

### 第五段：假线索（C2350-C2399）
- 第2350章：进入上古遗迹后发现是陷阱，遗书是假造的
- 读者此时的状态：被骗，和主角一起感受挫败
- 角色此时的状态：愤怒，但这次的目标是谁？不知道

### 第六段：真线索（C2400-C2600）← 当前正在写的段
- 第2400章：在陷阱的废墟中发现了真正的遗书——藏在墙体内
- 第2457章：[当前正在写] 按照真遗书的线索追踪到南海
- 读者此时的状态：重新燃起希望，但也在警惕——会不会又是假的？
- 角色此时的状态：警觉但不退缩

### 第七段：揭秘（C2601-C2700）
- 第2650章预期：找到真相——父母为保护一种功法而死的
- 第2700章预期：接受真相，支线完成
- 读者预期的状态：释然
- 角色预期的状态：释然，但某种力量传承给了他

## 本支线与其他支线的交叉点
| 交叉点 | 章 | 关联支线 | 关联方式 |
|--------|:---:|---------|---------|
| 玉佩的纹路 | C0080 | 支线B-王老 | 王老认出纹路 |
| 遗书被伪造 | C2350 | 支线D-魔教 | 魔教伪造遗书引诱主角 |
| 真遗书揭示的功法 | C2700 | 支线A-主线 | 功法是最终决战的关键 |

## 待写入的段
- [ ] C2457-C2600：南海追踪 → 现在正在写
- [ ] C2601-C2650：最终揭秘
- [ ] C2651-C2700：接受真相 + 传承

## 写作指南（给agent的提示）
- 追踪线索的节奏：每段20-30章，太密显得累，太疏读者会忘
- 假线索段（第五段）不要让读者觉得被玩弄——要让他们觉得「和我们一起被骗了」
- 揭秘段要留一手——就算揭示了真相，也可以留一个新的小问题
```

## 故事线操作协议

### 创建故事线

弧规划完成后，从 `arc-plan.md` 中提取所有识别到的叙事线：

```
识别规则：
- 有独立的目标→达成轨迹的 → 独立故事线
- 每条线必须有明确的：起点、终点、几个关键段
- 每条线标注：优先级(P0主线驱动/P1重要/P2点缀)
- 每条线标注：叙事类型(成长/解密/爱情/复仇/权力/悬疑/...)
```

### 每章更新故事线

每章创作完成后更新：

```
1. 在 storylines-overview.md 更新本线「当前段」和「最近章节」
2. 如果本章是某个段的结束/新段的开始 → 更新 storylines/{线名}.md
3. 如果本线在本章有重要发展 → 添加叙事路径条目
4. 如果本线和其他线在本章有交叉 → 更新两边的「交叉点」表
```

### 每卷更新故事线

每卷结束时审计：

```
□ 每条线的进度是否符合预期？
□ 有没有线被遗忘超过2卷(100章)？
□ 有没有线已经完成了但没有标记完成？
□ 线的优先级是否需要调整？（某条线变得更重要/不重要了）
```

### 续写预热时加载故事线

```
续写预热协议v3 — 在v2的10步之后新增：
11. 读 storylines-overview.md → 所有活跃线的状态
12. 读本章涉及的所有 storylines/{线名}.md → 线的完整叙事路径和情绪弧
13. 读 part-chapter-map.json → 本章的呼应关系和读者情绪目标
14. 如果最近50章内跨了一个篇章边界 → 重读上一篇章的 part-summary
```

## 篇章级操作协议

### 篇章规划（每个弧开始时生成）

```
弧规划 → 拆为3-5个篇章 → part-plan.md
每个篇章有：主题、起点状态、终点状态、核心冲突、
            关键事件时间线、读者情绪目标
```

### 篇章总结（每个篇章结束时生成）

```
篇章完成后：
1. 生成篇章故事总结（5000-8000字）→ part-summaries/
2. 逐条审计所有故事线在本篇章的进度
3. 列出遗留问题（交给后续篇章）
4. 评估读者体验
5. 如果偏航，调整下一个篇章的规划
```

```
3000章小说，典型故事线数量: 6-12条
```
