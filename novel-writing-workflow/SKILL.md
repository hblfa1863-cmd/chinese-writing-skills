# 小说创作编排工作流

> 编排 chinese-novelist（引擎）+ chinese-writing-skills（知识库）= 端到端的小说创作系统

## 执行入口

当用户说"开始写小说""创作一部小说""帮我写一部XX小说"时，按以下流程启动：

---

### 第 0 步：模式分派（按章节规模自动选择）

```
用户说出需求后 → 确定总章数
              │
              ├─ ≤50章  → 标准模式（第1-5步）
              │           一次性规划，上下文卸载，滚动规划可选
              │
              ├─ 51-299章 → 长篇模式
              │             启用滚动规划（每10章刷新）+ 上下文卸载
              │             可在卷结束时（每50章）暂停确认
              │
              └─ ≥300章 → 超长篇模式
                          弧/卷/章三层规划 + 人物状态机 + 伏笔管理 + 卷仪式
                          详见 references/ultra-long-novel-design.md
```

### 超长篇模式特殊流程（≥300章）

当检测到用户在创作 ≥300 章的小说时，执行以下特殊流程：

#### 0.A 弧级规划

1. 告知用户将使用超长篇模式
2. 与用户共同定义 6-10 个弧（每个弧 300-500 章）
3. 每个弧定义：起点 → 终点 → 核心冲突 → 关键转折点
4. 生成 `arc-plan.md`
5. 用户确认

#### 0.B 首卷规划

1. 选择弧1的前 50 章作为第一卷
2. 生成详细卷规划 `volume-plans/vol-01-XXX.md`
3. 初始化 `characters/` 目录的人物状态机文件
4. 初始化 `foreshadowing.json`
5. 初始化 `world-state.md`
6. 用户确认卷规划

#### 0.C 进入创作（见第3步"超长篇上下文装载"）

其他流程与长中篇相同，但每章创作时的上下文装载方式不同（见下文）。

---

### 第 1 步：启动引擎

通知用户：

> 📖 小说创作编排工作流已启动。
> 
> 我将同时调动两个子系统：
> - **创作引擎** (chinese-novelist)：管理创作流程、章节生成、字数校验、反AI检测
> - **技法知识库** (chinese-writing-skills)：在每个阶段提供对应写作技法指导
> 
> 首先，让我了解你想写一个什么样的故事。

然后进入 chinese-novelist 的 Phase 0-1 标准流程（初始化 + 三层递进式问答）。

---

### 第 2 步：规划阶段 — 技法注入

在 novelist 生成大纲和人物档案后，进入规划确认前，**自动挂载**对应的技法模块进行深度注入。

#### 2.1 读取技法模块

从 chinese-writing-skills 中读取以下模块，为规划提供技法支撑：

```
□ 01-人物塑造 → 检查人物档案是否有弧光、矛盾、侧面揭示
□ 02-情节设计 → 检查大纲冲突层次、节奏曲线、高潮布局
□ 08-爽点设计 → 检查爽点分布和密度（确保 writer JSON 有合理的爽点章标注）
□ 09-开篇技法 → 确认前 3 章标记为 opening，符合黄金三章标准
□ 07-悬念构建 → 检查总悬念、阶段悬念、章节悬念是否分层设置
□ 10-结尾技法 → 确认最后 1-2 章标记为 ending，结局类型明确
```

#### 2.1.5 验证 chapterType 标注

**在确认规划前，必须验证 novelist 生成的写作计划 JSON 中每章的 `chapterType` 标注是否合理**：

```
□ 第1-3章 chapterType 是否为 "opening"（如果没有，要求 novelist 修正）
□ 爽点章是否标注为 "pleasure"（如果没有，检查是否漏标或爽点分布有问题）
□ 反转章是否标注为 "twist"
□ 高潮章是否标注为 "climax"
□ 最后 1-2 章是否标注为 "ending"
□ 过渡章是否标注为 "transition"
□ chapterType 分布是否合理（不能全是 setup 或全是 pleasure）
```

#### 2.2 技法对照检查

对 novelist 生成的规划内容执行对照检查：

```
人物档案检查：
  □ 主角是否有明确的弧光（开始→结束的变化）？
  □ 主角是否有至少一处内在矛盾？
  □ 反派是否有自己的逻辑？是否是主角的"黑暗镜像"？
  □ 配角是否每个都有明确功能？

情节大纲检查：
  □ 每章是否至少设计了一个冲突？
  □ 冲突是否有层次叠加（人vs人 + 人vs自我 + ...）？
  □ 高潮赌注是否最大化？
  □ 爽点分布是否合理（蓄力→小爽→压抑→中爽→压抑→大爽）？

开篇检查：
  □ 前500字是否有强钩子？
  □ 黄金三章结构是否完整？
  □ 是否有天气/起床/吃早餐等致命错误？
```

#### 2.3 生成增强规划

根据技法检查结果，对规划提出优化建议（3-5条），展示给用户选择：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 技法增强建议（根据知识库分析）

1. 🔶 人物增强：主角弧光不够清晰
   当前：侦探查案找到凶手
   建议：增加"侦探自己也做过类似的事"的内心挣扎
   依据：人物塑造模块 §反派是主角的黑暗镜像

2. 🔶 爽点优化：第三章打完脸，第五章才第二个爽点
   建议：第四章增加一个小爆点（配角反水？新线索颠覆认知？）
   依据：爽点设计模块 §密度控制 — 你的题材适合每2-3章一个爽点

3. ✅ 开篇通过：第一章从争执开始，符合黄金三章标准

你希望应用这些建议后再开始创作，还是直接用当前规划？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 第 3 步：创作阶段 — 实时技法参考

进入 Phase 3 创作后，每章创作时**自动加载对应技法模块作为 Prompt 上下文**。

#### 超长篇模式（≥300章）的特殊上下文装载协议

> **核心原则**：不在上下文里堆累积物。上下文只放"地图"，详细信息去文件系统检索。

**每章开始前，从文件系统检索并装入上下文**：

```
[上下文装载清单 — 每章创作前执行]

1. 当前章大纲 → 从 volume plan 提取 (200 tokens)
2. 上章完整摘要 → 从上一章的收尾步骤写入 (300 tokens)
3. 本章出场角色状态 → 读 characters/{角色名}-state.md，仅提取相关字段 (200 tokens/角色)
4. 本章可回收的活跃伏笔 → 读 foreshadowing.json，筛选 (300 tokens)
5. 世界状态关联段落 → 读 world-state.md，提取与本章场景相关的段落 (200 tokens)
6. 当前卷总结 → 读 volume-summaries/vol-XX-summary.md (1000 tokens)
7. 当前弧总结 → 读 arc-summaries/arc-XX-summary.md (1000 tokens)
8. 总览摘要 → 读 arc-plan.md 的终极目标+当前进度段 (300 tokens)

总计装入: ~3,500-4,000 tokens（占上下文的 2%）
```

**每章完成后，更新文件系统**：

```
[上下文卸载 + 文件更新 — 每章创作后执行]

1. 本章摘要 → 追加到 volume summary
2. 人物状态变更 → 更新 characters/{角色名}-state.md
3. 新埋伏笔 → 注册到 foreshadowing.json
4. 回收伏笔 → 标记 revealed_chapter，移入 revealed 数组
5. 世界状态变更 → 更新 world-state.md
6. 清除本章正文 ← 从上下文删除
7. 保留状态索引在上下文：上章摘要 + 卷/弧总结 + 当前章大纲
```

#### 每章创作时的技法挂载（所有模式通用）

| chapterType | 自动加载的技法模块 | 用途 |
|:-----------:|-------------------|------|
| `opening` | 09-开篇技法 + 04-场景描写 + 01-人物塑造 | 黄金三章质量，建立人物记忆点 |
| `setup` | 02-情节设计 + 04-场景描写 + 05-对话设计 | 铺设新线索，引入新角色 |
| `pleasure` | 08-爽点设计 + 02-情节设计 §高潮 | 爽点蓄力-释放节奏 |
| `twist` | 02-情节设计 §转折 + 07-悬念构建 §信息差 | 转折有铺垫可回看 |
| `climax` | 02-情节设计 §高潮 + 12-情感描写 + 06-节奏控制 | 赌注最大化，弧光完成 |
| `transition` | 06-节奏控制 + 12-情感描写 + 03-语言风格 | 慢下来但不无聊 |
| `reveal` | 07-悬念构建 §伏笔回收 + 02-情节设计 §信息释放 | 揭示有力，既答又问 |
| `ending` | 10-结尾技法 + 12-情感描写 | 收得好，有余韵 |
| **所有章** | 03-语言风格 + 07-悬念构建 §钩子 | 行文基线 + 每章结尾钩子 |

#### 创作 Prompt 模板（每章创作时根据 chapterType 组装）

> **关键原则：技法约束必须是可验证的量化指标，不是模糊原则。agent 创作时直接将其作为 checklist 执行。**

```
[当前章节：第X章 / 共Y章]
[章节类型：从 02-写作计划.json 的 chapterType 字段读取 = "XXX"]
[本类型需加载的技法模块：按上表自动匹配]
[本章大纲：从 01-大纲.md 提取]
[上章摘要：从 01-大纲.md 的章节摘要区提取]
[人物状态快照：各角色最近状态]
[未回收伏笔清单：还在悬置的线索]

写作要求：
1. 遵循 chinese-novelist Phase 3 标准流程
2. 执行以下可验证的技法检查项（由 chapterType 决定）：

   === 按 chapterType 的量化约束（写入大纲后自动生效） ===

   [opening]
   □ 前500字有强钩子（冲突/悬念/反常规）
   □ 主角首次出场≥100字，包含相貌+动作+态度
   □ 第一章调动至少三感
   □ 最后一段必须有读者想知道"然后呢"的钩子

   [pleasure]
   □ 蓄力段（压迫/不公/委屈）≥800字
   □ 爆发段（反击/打脸/真相）≥500字
   □ 至少 1 个旁观者震惊反应（非主角视角）
   □ 爆发后≥200字"余震"（人物反应+情绪波动）

   [twist]
   □ 转折前≥2处可回看的铺垫
   □ 转折后≥300字后果展示（情绪/行动/关系变化）
   □ 转折必须影响至少 2 个角色的状态

   [climax]
   □ 赌注最大化（至少一个角色面对生死/毁灭/失去一切）
   □ 情感描写≥3处（恐惧→愤怒→释然等情绪层次推进）
   □ 冲突爆发段≥1000字
   □ 高潮后≥500字"落潮"（平静中的余韵）

   [transition]
   □ 小冲突≥1个（不超过主线的破坏性）
   □ 人物关系深化≥1处（对话/小动作/回忆中展示）
   □ 至少埋下 1 个新的小伏笔

   [reveal]
   □ 揭示的信息必须对之前≥2章产生回看效应
   □ 揭示同时提出≥1个新问题
   □ 揭示后≥300字展示角色的"重新理解世界"过程

   [ending]
   □ 回收≥90%的伏笔
   □ 展示"新常态"≥500字
   □ 结尾段必须和开头段有某种呼应
   □ 情感余韵≥200字（不平淡结束）

   === 所有 chapterType 共通约束 ===

   □ 禁用词扫描通过（check_ai_fingerprint.py）
   □ 句子长度标准差≥12，最短≤5字，最长≥60字
   □ 破折号≥3，省略号≥5，感叹号≥3
   □ 口语表达≥3处
   □ 闲笔（无用细节）≥3处
   □ 情绪模糊点≥2处
   □ 对话占比≥25%
   □ 刻意保留 2-3 处小瑕疵

3. 完成后：AI痕迹扫描（≥80%）→ 上下文卸载（保留摘要+状态+伏笔+下章大纲）

开始创作第X章。
```

### 第 4 步：校验阶段 — 技法对照审查

novelist Phase 4 校验完成后，增加「技法对照审查」步骤：

#### 4.1 逐章技法审查

对每章执行技法维度的评分：

```python
# 伪代码：技法审查维度
technique_review = {
    "第X章": {
        "人物": {
            "弧光推进": "✓ 主角在这一章首次承认自己的恐惧",
            "矛盾揭示": "✓ 展示了表面冷静 vs 内心焦虑"
        },
        "情节": {
            "冲突强度": "⭐⭐⭐ 人vs人（对峙）+ 人vs自我（内心挣扎）",
            "节奏曲线": "✓ 符合本章类型节奏模型"
        },
        "语言": {
            "金句": "✓ 1处（\"他不是不敢死，是不敢活\")",
            "句式变化": "✓ 标准差14.3，符合要求"
        },
        "对话": {
            "潜台词": "✓ 关键对话有双关层面",
            "辨识度": "⚠ 两个配角对话风格太接近"
        }
    }
}
```

#### 4.2 全局技法审查

```
全书技法质量报告：

人物维度：
  ✅ 主角弧光完整（从逃避→面对→牺牲）
  ✅ 反派有深度动机（不是因为"坏"才坏）
  ⚠️ 第7章配角功能模糊，建议要么删要么给一个明确功能

情节维度：
  ✅ 冲突层次丰富（人vs人 + vs社会 + vs自我）
  ✅ 转折有铺垫（回看第3、6、9章都有伏笔）
  ⚠️ 第12-14章节奏偏平，建议压缩或插入一个突发事件

语言维度：
  ✅ 平均金句密度适中（每2000字1.3处）
  ✅ 句子长度变化达标（全书标准差13.8）
  ⚠️ 第4章对话占比仅12%，偏叙述

爽点维度：
  ✅ 爽点分布符合"蓄力-释放"节奏
  ✅ 最终大爽点的情感五层递进完整
  ⚠️ 第18章的爽点释放太快，建议延长"反应"阶段

悬念维度：
  ✅ 四层悬念体系完整
  ✅ 每章结尾钩子类型多样（不只是"突然揭示"一种）
  ⚠️ 第15章的暗伏笔尚未回收

总评：⭐⭐⭐⭐（4.2/5）
```

#### 4.3 生成改进清单

```
📋 修订建议（按优先级）：

P0-必须改：
  □ 第7章：给配角李四一个明确的功能性（建议：成为第15章的叛徒）

P1-建议改：
  □ 第12-14章：压缩过渡，合并第13和14章的关键事件
  □ 第15章：回收第6章关于红鞋子的暗伏笔
  □ 第18章：延长爽点爆发后的"周围人反应"场景（至少增加200字）
```

## 卷完成仪式（超长篇模式 ≥300章）

每完成 50 章（一卷），强制执行卷完成仪式：

```
1. 生成本卷总结（1000-2000字）→ 写入 volume-summaries/vol-XX-summary.md
2. 更新所有活跃人物状态文件 → characters/*-state.md
3. 审计伏笔健康度 → 回收率 ≥ 40%
4. 检查弧级规划对齐度 → 是否偏离主线
5. 生成卷末审查报告 → quality-reports/vol-XX-review.md
6. 向用户展示摘要，确认后进入下一卷
```

## 弧完成仪式（超长篇模式 ≥300章）

```
1. 生成本弧完成总结（3000-5000字）→ arc-summaries/arc-XX-summary.md

3. 深度审查人物弧光完成度
4. 生成读者体验曲线分析
5. 向用户展示完整弧报告，确认后进入下一弧
```

## 迭代扩展指南

### 新增技法模块时

1. 在 chinese-writing-skills 中添加新的 `.md` 技法文件
2. 更新 chinese-writing-skills 的 SKILL.md 模块索引
3. 在本文件（novel-writing-workflow）的「第 3 步」中增加对应位置的挂载规则
4. 在「第 4 步」技法审查中增加对应维度的评分

### 新增小说题材时

1. 在 chinese-writing-skills 的 `references/genres/` 添加题材专项
2. 在本文件增加题材识别逻辑（从 Phase 1 的问答结果判断题材）
3. 根据题材调整技法挂载权重（如悬疑偏悬念模块、言情偏情感模块、玄幻偏爽点模块）

### 版本管理

```
chinese-novelist (引擎版本) + chinese-writing-skills (知识库版本) + novel-writing-workflow (编排版本)
      ↓                           ↓                                    ↓
   独立迭代                    独立迭代                        依赖上述两者的版本组合
```
