# 小说创作编排工作流

> 编排 chinese-novelist（引擎）+ chinese-writing-skills（知识库）= 端到端的小说创作系统

## 工作原理

```
chinese-writing-skills          novel-writing-workflow          chinese-novelist
   (技法知识库)                   (编排层/本 Skill)              (创作引擎)
        │                              │                            │
        ├─ 人物塑造 ────────────┐       │       ┌─ Phase 0: 初始化 ←┤

        ├─ 开篇技法 ────────┐ │ │       │   ┌───┤ Phase 1: 交互问答  │
        │                   ▼ ▼ ▼       │   │   │                    │
        │              ┌──────────┐     │   │   └────────────────────┘
        │              │ 按阶段   │     │   │   ┌────────────────────┐
        │              │ 自动挂载 │←────┼───┤   │ Phase 2: 规划确认   │
        │              │ 对应模块 │     │   │   │ + 技法深度注入      │
        │              └──────────┘     │   │   └────────────────────┘
        │                   │           │   │   ┌────────────────────┐
        ├─ 语言风格 ────────┤           │   │   │ Phase 2.5: 模式选择 │
        ├─ 场景描写 ────────┤           │   │   └────────────────────┘

        ├─ 爽点设计 ────────┤           │   │   └────────────────────┘
```

## 执行入口

当用户说"开始写小说""创作一部小说""帮我写一部XX小说"时，按以下流程启动：

---

### 第 1 步：启动引擎

通知用户：

> 📖 小说创作编排工作流已启动。
> 
> 我将同时调动两个子系统：
> - **创作引擎** (chinese-novelist)：管理创作流程、章节生成、字数校验、反AI检测
> - **技法知识库** (chinese-writing-skills)：在每个阶段提供对应写作技法指导
> 
> 首先，让我了解你想写一个什么样的故事。

然后进入 chinese-novelist 的 Phase 0-1 标准流程（初始化 + 三层递进式问答）。

---

### 第 2 步：规划阶段 — 技法注入

在 novelist 生成大纲和人物档案后，进入规划确认前，**自动挂载**对应的技法模块进行深度注入。

#### 2.1 读取技法模块

从 chinese-writing-skills 中读取以下模块，为规划提供技法支撑：

```
□ 01-人物塑造 → 检查人物档案是否有弧光、矛盾、侧面揭示
□ 02-情节设计 → 检查大纲是否有七种冲突层次、节奏曲线
□ 08-爽点设计 → 检查爽点分布是否合理（有没有爽点密度规划）
□ 09-开篇技法 → 确认第一章是否符合黄金三章标准
□ 07-悬念构建 → 检查总悬念、阶段悬念、章节悬念是否分层设置
```

#### 2.2 技法对照检查

对 novelist 生成的规划内容执行对照检查：

```
人物档案检查：
  □ 主角是否有明确的弧光（开始→结束的变化）？
  □ 主角是否有至少一处内在矛盾？
  □ 反派是否有自己的逻辑？是否是主角的"黑暗镜像"？
  □ 配角是否每个都有明确功能？

情节大纲检查：
  □ 每章是否至少设计了一个冲突？
  □ 冲突是否有层次叠加（人vs人 + 人vs自我 + ...）？
  □ 高潮赌注是否最大化？
  □ 爽点分布是否合理（蓄力→小爽→压抑→中爽→压抑→大爽）？

开篇检查：
  □ 前500字是否有强钩子？
  □ 黄金三章结构是否完整？
  □ 是否有天气/起床/吃早餐等致命错误？
```

#### 2.3 生成增强规划

根据技法检查结果，对规划提出优化建议（3-5条），展示给用户选择：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 技法增强建议（根据知识库分析）

1. 🔶 人物增强：主角弧光不够清晰
   当前：侦探查案找到凶手
   建议：增加"侦探自己也做过类似的事"的内心挣扎
   依据：人物塑造模块 §反派是主角的黑暗镜像

2. 🔶 爽点优化：第三章打完脸，第五章才第二个爽点
   建议：第四章增加一个小爆点（配角反水？新线索颠覆认知？）
   依据：爽点设计模块 §密度控制 — 你的题材适合每2-3章一个爽点

3. ✅ 开篇通过：第一章从争执开始，符合黄金三章标准

你希望应用这些建议后再开始创作，还是直接用当前规划？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

### 第 3 步：创作阶段 — 实时技法参考

进入 Phase 3 创作后，每章创作时**自动加载对应技法模块作为 Prompt 上下文**。

#### 每章创作时的技法挂载

| 章节位置 | 自动加载的技法模块 | 用途 |
|----------|-------------------|------|
| **第1章** | 09-开篇技法 + 04-场景描写 | 确保黄金三章质量，开头场景五感充沛 |
| **前3章** | 09-开篇技法 + 01-人物塑造 | 建立人物记忆点，确保钩子强度 |
| **普通章** | 03-语言风格 + 04-场景描写 + 05-对话设计 | 行文质量基线 |
| **爽点章** | 08-爽点设计 + 02-情节设计 §高潮构建 | 确保爽点爆发有力量 |
| **过渡章** | 06-节奏控制 + 12-情感描写 | 慢下来但不能无聊 |
| **反转章** | 02-情节设计 §转折艺术 + 07-悬念构建 §信息差 | 转折有力、伏笔回收 |
| **结局章** | 10-结尾技法 + 12-情感描写 | 收得好、有余韵 |
| **每章结尾** | 07-悬念构建 §章节结尾钩子 | 确保每章结尾有翻页冲动 |

#### 创作 Prompt 模板（示例）

```
[当前章节：第X章 / 共Y章]
[章节类型：普通章/爽点章/反转章/结局章]
[本章大纲：从写作计划中提取]
[上章摘要：300字]

写作要求：
1. 遵循 chinese-novelist Phase 3 标准流程（写前分析→撰写→润色→检查）
2. 同时遵循以下技法指导（来自 chinese-writing-skills）：

   📝 语言风格（03）：
   - 句子长度标准差 > 12
   - 至少1个≤5字短句 + 1个≥60字长句
   - 金句密度：本章至少1处

   🎬 场景描写（04）：
   - 本章关键场景至少调动三感（视觉+听觉+嗅觉）
   - 环境映射情绪（喜悦→光线亮暖 / 紧张→声音刺耳）

   💬 对话设计（05）：
   - 每段对话至少完成一项功能
   - 去掉"他说/她说"后能否辨识说话人？
   - 关键对话要有潜台词层面

   ⏱️ 节奏控制（06）：
   - 本章节奏曲线：开头快钩→中段起伏→高潮→回落→钩子

   [如果是爽点章，额外挂载 08-爽点设计]
   [如果是反转章，额外挂载 02-情节设计 §转折]
   [如果是最后3章，额外挂载 10-结尾技法]

3. 生成后执行 AI 痕迹扫描（chinese-novelist 标准流程）

开始创作第X章。
```

### 第 4 步：校验阶段 — 技法对照审查

novelist Phase 4 校验完成后，增加「技法对照审查」步骤：

#### 4.1 逐章技法审查

对每章执行技法维度的评分：

```python
# 伪代码：技法审查维度
technique_review = {
    "第X章": {
        "人物": {
            "弧光推进": "✓ 主角在这一章首次承认自己的恐惧",
            "矛盾揭示": "✓ 展示了表面冷静 vs 内心焦虑"
        },
        "情节": {
            "冲突强度": "⭐⭐⭐ 人vs人（对峙）+ 人vs自我（内心挣扎）",
            "节奏曲线": "✓ 符合本章类型节奏模型"
        },
        "语言": {
            "金句": "✓ 1处（\"他不是不敢死，是不敢活\")",
            "句式变化": "✓ 标准差14.3，符合要求"
        },
        "对话": {
            "潜台词": "✓ 关键对话有双关层面",
            "辨识度": "⚠ 两个配角对话风格太接近"
        }
    }
}
```

#### 4.2 全局技法审查

```
全书技法质量报告：

人物维度：
  ✅ 主角弧光完整（从逃避→面对→牺牲）
  ✅ 反派有深度动机（不是因为"坏"才坏）
  ⚠️ 第7章配角功能模糊，建议要么删要么给一个明确功能

情节维度：
  ✅ 冲突层次丰富（人vs人 + vs社会 + vs自我）
  ✅ 转折有铺垫（回看第3、6、9章都有伏笔）
  ⚠️ 第12-14章节奏偏平，建议压缩或插入一个突发事件

语言维度：
  ✅ 平均金句密度适中（每2000字1.3处）
  ✅ 句子长度变化达标（全书标准差13.8）
  ⚠️ 第4章对话占比仅12%，偏叙述

爽点维度：
  ✅ 爽点分布符合"蓄力-释放"节奏
  ✅ 最终大爽点的情感五层递进完整
  ⚠️ 第18章的爽点释放太快，建议延长"反应"阶段

悬念维度：
  ✅ 四层悬念体系完整
  ✅ 每章结尾钩子类型多样（不只是"突然揭示"一种）
  ⚠️ 第15章的暗伏笔尚未回收

总评：⭐⭐⭐⭐（4.2/5）
```

#### 4.3 生成改进清单

```
📋 修订建议（按优先级）：

P0-必须改：
  □ 第7章：给配角李四一个明确的功能性（建议：成为第15章的叛徒）

P1-建议改：
  □ 第12-14章：压缩过渡，合并第13和14章的关键事件
  □ 第15章：回收第6章关于红鞋子的暗伏笔
  □ 第18章：延长爽点爆发后的"周围人反应"场景（至少增加200字）
  □ 第4章：增加至少500字对话，将对话比提升至≥20%

P2-锦上添花：
  □ 第3章结尾钩子可从"突发揭示"改为"致命选择"，读者卷入感更强
  □ 整体：考虑给每个配角增加一个"习惯性动作"来提升辨识度
```

## 迭代扩展指南

### 新增技法模块时

1. 在 chinese-writing-skills 中添加新的 `.md` 技法文件
2. 更新 chinese-writing-skills 的 SKILL.md 模块索引
3. 在本文件（novel-writing-workflow）的「第 3 步」中增加对应位置的挂载规则
4. 在「第 4 步」技法审查中增加对应维度的评分

### 新增小说题材时

1. 在 chinese-writing-skills 的 `references/genres/` 添加题材专项
2. 在本文件增加题材识别逻辑（从 Phase 1 的问答结果判断题材）
3. 根据题材调整技法挂载权重（如悬疑偏悬念模块、言情偏情感模块、玄幻偏爽点模块）

### 版本管理

```
chinese-novelist (引擎版本) + chinese-writing-skills (知识库版本) + novel-writing-workflow (编排版本)
      ↓                           ↓                                    ↓
   独立迭代                    独立迭代                        依赖上述两者的版本组合
```
